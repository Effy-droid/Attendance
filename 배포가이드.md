# Vercel 배포 가이드 — 자동 SMS 발송 시스템

> 학원장님이 클릭만 하시면 되도록 만들었어요.
> 총 소요 시간: **30~45분** (영업일 1~2일 알리고 발신번호 승인 포함 안 됨)
> 모든 단계가 **무료**입니다.

## ✅ Step 0: 준비물 체크리스트

배포 전 다음이 완료되어야 합니다:

- [ ] 알리고 발신번호 **승인 완료** (영업일 1~2일 후 메일로 알림 옴)
- [ ] 알리고 **API Key 발급** (콘솔 → API 사용 신청 → 발급)
- [ ] 알리고 **User ID** 메모 (가입 시 만든 ID)
- [ ] 알리고 **충전 완료** (5만원 권장)

> ⚠️ 발신번호 승인 전에 배포해도 됩니다. 단, 실제 SMS 발송은 승인 후부터 작동.

## 🚀 Step 1: GitHub 가입 + 코드 업로드 (10~15분)

Vercel은 GitHub와 연결되어야 자동 배포가 가능해요. GitHub 계정이 없으면 만들어야 합니다.

### 1-1. GitHub 가입 (5분)

1. **[github.com](https://github.com)** 접속
2. 우측 상단 **"Sign up"** 클릭
3. 이메일·비밀번호·사용자명 입력 (사용자명은 영문, 예: `kanghaejee`)
4. 본인 인증 (이메일 코드)
5. **Free** 플랜 선택 (개인용 무료)

### 1-2. 새 저장소(Repository) 만들기 (3분)

1. GitHub 로그인 후 우측 상단 **"+"** → **"New repository"**
2. 입력:
   - **Repository name**: `attendance-app` (또는 원하는 이름)
   - **Description**: (선택) "학원 출결 자동 알림"
   - **Public** 또는 **Private** 선택 — **Private 권장** (학원 정보 비공개)
   - ✅ "Add a README file" 체크
3. **"Create repository"** 클릭

### 1-3. 파일 업로드 (5~7분)

1. 만든 저장소 페이지에서 **"Add file"** → **"Upload files"** 클릭
2. **`바탕 화면/attendance/attendance-vercel/` 폴더의 모든 파일을 드래그**:
   - `index.html`
   - `vercel.json`
   - `package.json`
   - `api/` 폴더 (안에 `send.js`, `balance.js`)
3. 폴더 구조가 유지되어야 함:
   ```
   attendance-app/
   ├── index.html
   ├── vercel.json
   ├── package.json
   └── api/
       ├── send.js
       └── balance.js
   ```
4. 하단 **"Commit changes"** 클릭

### ⚠️ api/ 폴더 업로드 안 될 때

GitHub 웹 UI는 폴더 자체 드래그 지원이 가끔 막혀요. 그러면:
1. `api/` 안의 파일을 먼저 업로드: 저장소 메인 → "Add file" → "Create new file"
2. 파일 이름란에 `api/send.js` 입력 (슬래시가 폴더 자동 생성)
3. 내용: 학원장님 PC의 `api/send.js` 내용 복사 → 붙여넣기
4. Commit
5. `api/balance.js`도 같은 방식으로

## 🚀 Step 2: Vercel 가입 + 배포 (5~10분)

### 2-1. Vercel 가입 (3분)

1. **[vercel.com](https://vercel.com)** 접속
2. **"Sign Up"** 클릭
3. **"Continue with GitHub"** 선택 (GitHub 계정으로 로그인)
4. 권한 승인 → 가입 완료
5. 무료 **Hobby** 플랜 자동 (신용카드 등록 불필요)

### 2-2. 프로젝트 임포트 (3분)

1. Vercel 대시보드 → **"Add New..."** → **"Project"**
2. GitHub 저장소 목록에서 `attendance-app` 찾기
3. **"Import"** 클릭
4. 설정 화면:
   - **Project Name**: 그대로 (또는 변경)
   - **Framework Preset**: "Other" (자동 감지됨)
   - **Root Directory**: 그대로 (`./`)
5. ⚠️ **아직 "Deploy" 버튼 누르지 마세요!** Step 2-3 환경변수 먼저 추가.

### 2-3. 환경변수 입력 ⚠️ 가장 중요 (5분)

배포 화면 하단의 **"Environment Variables"** 섹션 펼치기. 다음 4~5개 추가:

| Key (이름) | Value (값) | 설명 |
|---|---|---|
| `ALIGO_API_KEY` | (알리고에서 발급받은 API Key) | 알리고 콘솔 → API 메뉴 |
| `ALIGO_USER_ID` | (알리고 가입 ID) | 알리고 가입 시 만든 ID |
| `ALIGO_SENDER` | (알리고 등록 발신번호) | 예: `01012345678` 또는 `0212345678` |
| `AUTH_TOKEN` | (학원장님이 정한 비밀 문자열) | 예: `attendance-secret-2026` (긴 랜덤 문자열 권장) |
| `ALIMTALK_TEMPLATE_CODE` | (선택, 알림톡 승인 후) | 빈 칸으로 두고 나중에 추가 |

> 💡 **AUTH_TOKEN 만들기 팁**: 학원장님이 직접 정한 비밀 문자열. 길고 복잡할수록 안전.
> 예: `myacademy-2026-xK9p3mZ` 또는 `행복학원_보안토큰_8472`
> 이 값을 잘 적어두세요. 키오스크에 같은 값을 입력해야 합니다.

각 환경변수 입력 후 **"Add"** 클릭.

### 2-4. 배포 시작 (1분)

환경변수 모두 입력했으면:

1. 화면 하단 **"Deploy"** 버튼 클릭
2. 배포 진행 (1~2분)
3. **🎉 배포 성공 화면**에 URL 표시 (예: `attendance-app-xxx.vercel.app`)

이 URL을 메모하세요. **이게 학원장님 키오스크에 입력할 API URL입니다.**

## 🚀 Step 3: 키오스크에 연결 (5분)

### 3-1. 키오스크 URL 접속

학원장님 iPhone 또는 PC에서 위에서 받은 Vercel URL 접속 (예: `attendance-app-xxx.vercel.app`)

→ 우리 앱이 보임. (Step 1에서 업로드한 `index.html`이 메인 페이지)

### 3-2. iPhone에 "홈 화면에 추가"

1. iPhone Safari에서 위 URL 접속
2. 하단 **공유** 버튼 (네모+화살표) 클릭
3. **"홈 화면에 추가"** 선택
4. 이름 입력 후 **"추가"**
5. iPhone 홈 화면에 앱 아이콘 생성 ✓

### 3-3. 자동 발송 모드 설정

1. 앱 우측 상단 **⚙️** 아이콘 → 관리자 PIN `8888` 입력
2. **🚀 자동발송** 탭 클릭
3. **자동 발송 토글 ON**
4. 입력:
   - **API URL**: `https://attendance-app-xxx.vercel.app` (Vercel 배포 URL, **앞에 `https://` 포함**)
   - **API 토큰**: 위 Step 2-3에서 정한 `AUTH_TOKEN` 값과 동일
   - **발송 방식**: SMS (알림톡 승인 전까지 SMS만)
5. **"자동발송 설정 저장"** 클릭
6. **"🧪 연결 테스트"** 클릭

### ✅ 연결 성공하면

```
✓ 연결 성공! 알리고 잔여: SMS XXXX건 / LMS XXXX건 / 잔액 XXXXX원
```

이런 메시지가 표시되면 **준비 끝!**

### ❌ 연결 실패하면

자주 있는 원인:

| 오류 메시지 | 원인 | 해결 |
|---|---|---|
| "401 Unauthorized" | AUTH_TOKEN 불일치 | 키오스크 토큰과 Vercel 환경변수의 AUTH_TOKEN이 정확히 같아야 함 |
| "500 알리고 자격증명 없음" | 환경변수 미설정 | Vercel → Settings → Environment Variables 다시 확인 |
| "연결 오류 (네트워크)" | URL 오타 | `https://`로 시작하는지, 끝에 `/` 없는지 확인 |
| "잔액 조회 실패" | 알리고 API Key 오류 | 알리고 콘솔에서 API Key 재발급 후 Vercel 환경변수 갱신 |

## 🚀 Step 4: 첫 발송 테스트 (5분)

진짜 SMS가 가는지 확인:

1. 관리자 → **🧒 학생 관리** → 새 학생 추가
2. 입력:
   - 이름: `테스트`
   - 부모 핸드폰: **본인 핸드폰 번호** (학원장님 본인)
   - PIN: 자동 생성됨 (본인 폰 뒤 4자리)
3. **저장** → 학생 목록에 추가 확인
4. **키오스크로 ✕** 클릭하여 메인 화면 복귀
5. **방금 그 PIN 입력 → 전송** 누르기
6. 화면에 "⏳ 발송 중..." → "✓ 등원 완료!" 표시
7. **본인 핸드폰에 1~3초 안에 SMS 도착**

🎉 도착했으면 자동화 시스템 완성!

## 🚀 Step 5: 학원 운영 시작

### 학생 등록
- CSV 일괄 등록 또는 한 명씩 추가
- iPhone Safari에서 작업 가능 (홈 화면에 추가한 앱)
- 학생 정보는 **iPhone Safari의 localStorage**에 저장됨

### 키오스크 모드 운영
1. iPhone을 학원 입구에 거치 (다이소 거치대)
2. 충전 케이블 항상 연결
3. **자동 잠금 OFF**: 설정 → 디스플레이 → 자동 잠금 → 안 함
4. **가이드 액세스 ON**: 설정 → 손쉬운 사용 → 가이드 액세스
5. Safari로 앱 열기 → 측면 버튼 3번 → 가이드 액세스 시작 → 학생들이 다른 앱 못 만짐

### 매일 학원장님이 할 일
- **거의 없음!** 자동 발송이라.
- 가끔 ⚙️ → 오늘 현황에서 발송 통계 확인
- 발송 실패 학생 있으면 (드물게) 재시도 또는 수동 발송
- 알리고 잔액이 1만원 미만 되면 충전

## 🚨 자주 발생하는 문제

### Q1: Vercel 배포가 실패해요
- Vercel 대시보드 → Deployments → 실패한 배포 클릭 → 로그 확인
- 보통 `package.json` 누락 또는 `vercel.json` 형식 오류
- 해결: 학원장님 PC의 파일과 GitHub의 파일이 정확히 같은지 확인

### Q2: 키오스크에서 "발송 실패"가 자주 떠요
- 알리고 잔액 부족 가능성 → 잔액 조회 버튼으로 확인
- 발신번호가 알리고에 등록한 것과 정확히 일치하지 않음 (예: `0212345678` vs `02-1234-5678`)
- 알리고 시스템 일시 점검 → 잠시 후 재시도

### Q3: 부모님이 SMS를 못 받았다고 해요
- 부모 번호가 정확한지 확인 (학생 관리에서 수정)
- 부모님 폰의 스팸 차단 설정 확인 (학원 발신번호를 화이트리스트에)
- 알리고 콘솔에서 발송 이력 조회 (실제로 발송 성공인지 확인)

### Q4: GitHub·Vercel 비밀번호를 잊었어요
- 각 사이트에서 "비밀번호 재설정" 진행
- 학원 운영 정보 노출 가능성 있으니 비밀번호 관리자(예: 1Password, 비트워든) 사용 권장

### Q5: 다른 사람이 우리 학원 앱 URL에 접속하면?
- URL을 알면 누구나 접속 가능 (무작위 추측은 어려움)
- 학생 정보는 각 기기 localStorage에만 → 다른 기기에선 빈 앱
- API는 AUTH_TOKEN으로 보호 → 토큰 모르면 SMS 발송 불가
- **추가 보안 원하면**: Vercel → Settings → Deployment Protection → "Vercel Authentication" 켜기 (학원장 본인만 접속 가능)
