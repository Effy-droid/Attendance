# 🎓 학원 출결 자동 알림 시스템

> **비개발자 학원장이 Claude Code로 만든 첫 번째 프로젝트**
>
> Anthropic에서 배운 것을 실전에 적용해보며,
> Voice Dump · Vibe Coding · Deep Interview 방법론으로
> 명세 → 합의 → 코딩 → 배포까지 직접 경험한 기록

🌐 **Live**: <https://attendance-pi-green.vercel.app/>

## 👋 들어가며

안녕하세요. 저는 학원을 운영하는 **학원장**이에요. 그리고 코딩을 단 한 줄도 해본 적이 없습니다.

이 저장소는 단순한 출결 앱 코드가 아니에요. **비개발자인 제가 AI(Claude Code)와 함께 진짜 앱을 처음부터 만들어 인터넷에 띄우기까지의 과정을 기록한 노트**입니다.

평소 학원에서 가장 번거로웠던 일을 직접 자동화하고 싶다는 생각, 그리고 Anthropic을 통해 알게 된 도구들을 연습 삼아 적용해보고 싶다는 마음에서 시작했어요.

## 💭 왜 이 프로젝트를 시작했나

### 매일 반복되던 작은 불편

학원을 운영하다 보면 매일 이런 일이 있어요:

> 학생이 도착했는지 부모님이 궁금해하심
> → 학원장이 학부모께 일일이 카톡으로 "○○ 도착했어요" 보냄
> → 매일 30~50명, 매일 30분~1시간 소요
> → 정작 수업 준비가 늦어짐

"이거 자동화되면 좋겠다"는 생각만 1년 넘게 했어요. 외주를 맡기자니 비용이 부담스럽고, 직접 만들자니 불가능해 보였죠.

### Claude Code를 만나고

그러던 중 **Anthropic의 Claude Code**를 접했어요. AI가 코딩을 도와주는 정도가 아니라, **AI와 대화하면서 진짜 앱이 만들어진다**는 걸 보고 충격받았어요.

> "비개발자인 내가 진짜로 만들 수 있을까?"

이 질문 하나가 이 프로젝트의 시작이었습니다. **연습 삼아, 그리고 진짜로 학원에서 쓸 수 있는 것을 만들어보자.**

## 🛠️ 적용한 방법론

이번 프로젝트에서 시도해본 4가지 접근법입니다.

### 1. 🎤 Voice Dump (음성 덤프)

처음엔 정리되지 않은 생각을 그냥 **음성으로 쏟아냈어요**:

> "일단내가만들고싶은거는 우리학원아이들이출석을했을때 아이들이부모님핸드폰뒷번호나 설정한번호네자리만누르면 부모님핸드폰으로자동으로 카카오톡이나메신저나어떤알림이가게끔만드는 어플을만들고싶어..."

띄어쓰기도 엉망, 문법도 엉망, 정리도 안 된 생각의 흐름. 그런데 이걸 그대로 AI에게 던졌더니 **AI가 핵심을 잡아내서 정리**해주더라고요.

→ **깨달음**: 비개발자는 깔끔한 문서를 쓸 필요가 없다. AI가 정리한다. 중요한 건 **머릿속의 진짜 의도를 그대로 꺼내는 것**.

### 2. 🔍 OMC의 Deep Interview (소크라테스식 인터뷰)

[oh-my-claudecode (OMC)](https://github.com/yeachan-heo/oh-my-claudecode)의 `/deep-interview` 슬래시 커맨드로 시작했어요.

AI가 한 번에 한 가지씩, 7라운드에 걸쳐 핵심 질문을 던졌어요:

| Round | 질문 | 모호도 |
|---|---|---|
| 1 | 알림 채널은? (SMS / 카톡 / 자체 앱) | 100% → 73.5% |
| 2 | 4자리 PIN은 무엇? | → 66% |
| 3 | 등원만? 등원+하원? | → 58% |
| 4 | 입력 단말기는? | → 48.5% |
| 5 | 치명적 실패 시나리오는? | → 32.1% |
| 6 | 학원 규모는? | → 28.2% |
| 7 | 메시지 형식은? | → **17.3%** ✅ |

매 라운드마다 **"모호도(ambiguity)"라는 점수가 수치로 표시**됐어요. 100%에서 시작해 17.3%까지 떨어지면 "충분히 명확해졌다"는 임계치 통과로 자동 명세서가 생성됐죠.

→ **깨달음**: 막연한 아이디어를 **수학적으로 측정 가능한 명세**로 바꿔주는 도구가 있다는 게 신기했어요. 평소 같으면 "그냥 만들어보자"로 시작했을 텐데, 이번엔 시작 전에 **무엇을 만들지가 명확**해졌어요.

### 3. 🎯 Ralplan 합의 계획 (Planner / Architect / Critic)

명세서가 나온 다음엔 OMC의 `/ralplan`으로 합의 계획서를 만들었어요. AI가 3명의 역할을 번갈아 수행했어요:

```
[Planner]      "이렇게 만들면 어떨까요? 11.5일짜리 계획입니다"
       ↓
[Architect]    "잠깐, 이 부분에 12개 위험요소가 있어요"
       ↓
[Critic]       "10개는 반드시 수정해야 합니다 (ITERATE)"
       ↓
[Planner R2]   "수정했어요. v2 입니다"
       ↓
[Architect R2] "이번엔 통과 (PASS)"
       ↓
[Critic R2]    "APPROVE (8.2점/10점)"
       ↓
       ✅ 합의 완료 → 코딩 시작
```

이 과정을 **2 라운드** 반복하면서 처음 5.5점이던 계획이 8.2점까지 올라갔어요. 단순한 코드 작성이 아니라 **위험 요소를 미리 발견하고 합의된 계획**으로 시작한다는 점이 인상적이었어요.

→ **깨달음**: AI 한 명이 결정하는 게 아니라, **여러 관점이 서로 비판하고 합의**하는 과정 자체가 품질을 만드는 거였어요.

### 4. 🌊 Vibe Coding (의도 기반 코딩)

계획이 합의되니, AI에게 "이 계획대로 만들어줘"라고만 했어요.

한 번의 메시지로 **1968줄의 코드**가 쏟아져 나왔고, 저는:
- 코드를 손으로 짜지 않았어요
- 변수 이름·함수 구조·CSS를 직접 신경 쓰지 않았어요
- 그저 "이게 잘 작동하는지", "내 의도와 맞는지"만 체크했어요

요즘 말하는 **"Vibe Coding"** — 코드를 짜는 게 아니라 **의도를 전달하고 검토만** 하는 방식이에요.

→ **깨달음**: 비개발자에게 가장 큰 진입장벽이었던 **"코드 문법 학습"**을 건너뛸 수 있다는 게 진짜였어요. 단, 의도를 정확히 전달하는 능력 (이전 단계에서 만든 명세서·계획서)이 그만큼 중요해졌어요.

## 🏗️ 만들어진 시스템

### 사용자 흐름

```
[학생이 학원 도착]
       ↓
[입구 iPhone 키오스크]
       ↓ 4자리 PIN 입력 + 전송
       ↓
[Vercel Serverless API]  ← GitHub와 연동된 자동 배포
       ↓ 알리고 API 호출
       ↓
[Aligo SMS/알림톡 게이트웨이]
       ↓ KT/SKT/LGU+ 통신망
       ↓
[부모님 핸드폰]  📱 "○○ 학생이 도착했습니다"
```

### 기술 스택

| 영역 | 선택 | 이유 |
|---|---|---|
| Frontend | Vanilla HTML/CSS/JS (단일 파일) | 비개발자가 한눈에 이해 가능 |
| Database | Browser localStorage | 별도 DB 가입 불필요, 무료 |
| Hosting | Vercel Hobby Tier | 무료, GitHub 자동 배포 |
| Serverless | Vercel Functions (Node.js) | 알리고 API 게이트웨이 |
| SMS API | Aligo (aligo.in) | 한국 SMS 1위, 알림톡 연동 가능 |
| Version Control | GitHub Private Repo | 무료, 학원 정보 비공개 |

### 운영비 (예상)

| 항목 | 월 비용 |
|---|---|
| Vercel + GitHub | 0원 |
| 알리고 SMS (학생 50명 기준) | 약 2~3만원 |
| **합계** | **약 2~3만원/월** |

## ✨ 주요 기능

**키오스크 (학생용)**
- 4자리 PIN 입력 → 자동 알림 발송
- 형제자매 PIN 충돌 시 학생 선택 화면
- 30분 throttle (중복 입력 차단)
- iPhone PWA로 풀스크린 작동

**관리자 (학원장용)**
- 학생/학부모 정보 관리 (CSV 일괄 등록 지원)
- 부모 핸드폰 뒤 4자리 자동 PIN 생성
- 출결 로그 (일/주/월 필터, CSV 내보내기)
- 메시지 템플릿 편집기 (변수 + 글자수 + 비용 미리보기)
- 발송 통계 + 실패 자동 재시도
- JSON 백업/복구

**보안**
- AUTH_TOKEN 기반 API 인증
- API Key는 Vercel 환경변수에만 저장 (브라우저 노출 안 됨)
- HTTPS 강제 + Private Repo

## 🤔 솔직한 회고 — 진짜로 어땠나

### 진짜 좋았던 것

1. **막연하던 아이디어가 수치화됨**
   - 모호도 100% → 17.3%라는 점수가 보이니까 진척 체감됨
   - "이만큼 명확해졌으니 시작해도 된다"는 객관적 기준

2. **여러 AI 관점의 합의 과정**
   - Planner 혼자 결정 vs Critic이 비판하고 다시 수정
   - 후자가 명백히 품질이 높았음

3. **외부 서비스 셋업도 같이 안내**
   - 알리고 가입 단계마다 화면 캡처 → AI 안내 → 다음 단계
   - 평소 같으면 막혔을 부분들을 함께 헤쳐나감

4. **AI가 직접 git push까지 해줌**
   - 평소 같으면 GitHub Desktop 같은 GUI 도구 필요했을 것
   - "너가 push 해줘" 한 마디로 끝남

### 진짜 어려웠던 것

1. **너무 많은 옵션 선택지**
   - "Cloudflare Workers / Vercel / Supabase / Lovable..."
   - 비개발자는 뭐가 뭔지 모름. AI 추천을 신뢰해야 했음

2. **외부 서비스 가입의 행정적 대기**
   - 알리고 발신번호 등록 = 영업일 1~2일
   - 사업자 인증 = 담당자 확인 대기
   - 이건 AI도 줄여줄 수 없음

3. **"이 앱을 운영비 들여야 한다"는 결심**
   - 월 2~5만원 운영비 발생
   - 하지만 매일 30분~1시간 절약되니 가치 있다고 판단

4. **모르는 영어 용어들**
   - Environment Variables, CORS, Serverless...
   - AI가 매번 한국어로 설명해주긴 하지만 학습 곡선 있음

### 가장 큰 깨달음

> **"비개발자에게 진짜 진입장벽은 코딩이 아니라, 무엇을 만들지 명확하게 표현하는 능력이다."**

코드는 AI가 짜준다. 어디서 막히는지도 AI가 알려준다. 외부 서비스 셋업도 AI가 안내한다.

남는 것은 **"내가 진짜 원하는 게 뭔가?"**라는 질문이고, 이걸 명확히 하는 데 Deep Interview·Ralplan 같은 도구들이 굉장히 큰 역할을 했어요.

## 🔧 코드 수정 + 자동 배포

이 저장소는 **Vercel과 자동 연동**되어 있어요.
GitHub에 push하면 1~2분 내 자동 배포됩니다:

```bash
git add .
git commit -m "수정 내용"
git push origin main
# → Vercel이 감지 → 자동 배포 → Live URL 갱신
```

비개발자에게도 이게 가능했던 이유는: **AI가 git 명령어까지 직접 실행**해줬기 때문이에요.

1. **음성으로 막연한 아이디어 쏟아내기** — 정리하지 마세요. 그대로 AI에게 던지세요.
2. **Deep Interview 같은 도구 써보기** — 모호함을 수치로 줄여줍니다.
3. **외부 서비스 가입까지 AI에게 안내받기** — "막힌 화면 캡처 → AI" 무한 반복.
4. **운영비를 미리 각오하기** — 월 2~5만원 정도. 하지만 절약되는 시간 가치가 더 큽니다.
5. **"못한다"는 가정을 버리기** — 저도 못할 거라고 생각했어요. 단 하루 만에 인터넷 배포까지 했습니다.
